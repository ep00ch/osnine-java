7   /**
8    * Gate Array at $A7E4 to $A7E7.
9    * The Gate Array double-register A7E4 and A7E5 allows you to read the Gate
10   * Array internal counters used for video generation. The register is only
11   * updated when the lightpen detects the raster beam.
12   * <dl>
13   * <dt>The 0xA7E4 register is made of the following bits:</dt>
14   * <dd>Bits 15 to 6: GPL counter. This is incremented every 8 octets in the
15   * visible area.</dd>
16   * <dd>Bits 5 to 3: Octet in current segment</dd>
17   * <dd>Bits 2 to 0: 1MHz, 2MHz and 4MHz clocks to specify bit.</dd>
18   * </dl>
19   * It takes one micro-second to paint one byte of pixels (i.e. 8 pixels).
20   * During that time the 1MHz, 2MHz and 4MHz clocks go from 0 to 1 to 0 at
21   * their respective frequencies. The 1 MHz can tell you which half of the
22   * byte is being painted, the 2 MHz which half of the half and so on.
23   * <dl>
24   * <dt>0xA7E6: Line counter</dt>
25   * <dd>bit 7: TL3, mirror bit of the counter above.</dd>
26   * <dd>bit 6: INILN: 0 when the GA is drawing vertical borders on the left and
27   * right of the screen.</dd>
28   * <dt>0xA7E7</dt>
29   * <dd>INITN bit 7: 0 when the GA is drawing horizontal borders above and below
30   * the screen. When both INITN and INILN are set, the GA is drawing pixels.</dd>
31   * </dl>
32   * <h2>Light pen algorithm</h2>
33   * <p>
34   * The algorithm waits for the beam to get under the active area by looking for
35   * INITN to go from 0 to 1 to 0. It then activates FIRQ signalling for CA1 in
36   * the PIA and sets 0xA7E4 to 1.
37   *
38   * As soon as the 6809 detects the possibility of using the light pen, it
39   * will use a special routine. The latter will set the D0 bit to the $A7E4
40   * address of the gate-array, which will validate the CKLP if it occurs.
41   */

57      /**
58       * Constructor.
59       * The screen memory is always 8192 bytes, but only uses 8000.
60       * A7E6 bit 6 is set when the beam is in the "active horizontal area"
61       * A7E7 bit 7 does the same for the "active vertical area".
62       *
63       * @param screen - Connection to the GUI
64       */
65      public GateArray(Screen screen) {
66          super(0xA7E4, 0xA7E4 + 4);
67          this.screen = screen;
68          memory = new int[SIZE];
69          for (int i = 0; i < SIZE; i++) memory[i] = 0;
70          memory[2] = 0x40;
71          memory[3] = INITN + 0x40;
72      }

74      @Override
75      protected int load(int addr) {
76          int relAddr = addr - getStartAddress();
77          memory[3] ^= INITN;
78          LOGGER.debug("Load: {} - {}", relAddr, memory[relAddr] & 0xFF);
79          return memory[addr - getStartAddress()] & 0xFF;
80      }

85          LOGGER.debug("Store: {} val: {}", relAddr, val);
86          if (relAddr == 0) {
87              screen.setIncrustation(val == 1);
88          }

94       * Store the lightpen coordinates in the gate array. On a real TV screen
95       * the beam paints 312 lines, of which 200 lines are visible. The vertical
96       * coordinate must therefore be offset by 56 rows to match. Horizontally
97       * the beam paints 512 pixels (320 visible). The X coordinate must be
98       * offset by 96 pixels.
99       * @param x - horisonal coordinate 0 - 319
100      * @param y - vertical coordinate 0 - 199
101      * FIXME: Make it work.

103     void setLightpenXY(int x, int y) {
104         if (x != -1) {
105             y += 56;
106             x += 96;
107             int inx = (y * 64 + (x / 8)) << 3;
108             memory[0] = (inx >> 8) & 0xFF;
109             memory[1] = (inx & 0xF8) | (x % 8);
110             memory[2] &= 0x7F | ((memory[1] << 1) & 0x80);
111         } else {
112             memory[0] = 0;
113             memory[1] = 0;
114             memory[2] = 0;
115         }
